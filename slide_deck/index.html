Why Crystal?

We all appreciate Ruby, or we wouldn't be here, right?

Some of the benefits of Ruby:

  • blocks
  • user-friendly syntax
  • versatile
  • useful sprinkles of functional programming
  • dynamic typing generally stays out of the way
  • strong tooling and testing ecosystem

Why Crystal?

Some of the drawbacks of Ruby:

  • difficult binary distribution story
  • raw performance is mediocre
  • concurrency can be an adventure
  • static type checking

    • competing standards

fib = Fiber.new do
  fib_seq = Hash.new do |h, k|
    h[k] = k < 2 ? k : h[k - 2] + h[k - 1]

  count = 1
  loop do
    Fiber.yield fib_seq[count]
    count += 1
  end
end

What Are The Differences?

Crystal specs look like this:

describe OpenTelemetry::Event, tags: ["Event"] do
  it "can create an event with direct initialization" do
    event = OpenTelemetry::Event.new(
      name: "test_event",
      attributes: {
        "test_attribute" => "test_value",
      }
    )
    event.name.should eq "test_event"
    event["test_attribute"].should eq "test_value"
  end

  it "can create an event with block based initialization" do
    event = OpenTelemetry::Event.new do |e|
      e.name = "test_event"
      e.attributes = {
        "test_attribute" => "test_value",
      }
    end
    event.name.should eq "test_event"
    event["test_attribute"].should eq "test_value"
  end
end
